# Preface

The Developer Handbook is a living engineering knowledge base built to improve technical judgment, deepen systems understanding, and reduce waste in daily software development.

It is not a command reference, a transcript of conversations, or a collection of disconnected notes. It is a maintained body of knowledge that evolves as practical experience grows.

The handbook begins with fundamentals because advanced tools become easier to understand when their underlying systems are clear. Unix, filesystems, processes, shells, Git, networking, automation, containers, cloud infrastructure, system design, debugging, and performance are treated as one connected system.

The governing principle is:

> Understand systems. Build mental models. Automate everything worth repeating.

The source of truth is the Markdown in this repository. Published documents, websites, PDFs, and other formats are generated from that source.
